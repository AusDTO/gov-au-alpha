Component Naming Conventions

For each component in app/assets/css/components/:

  • .module-name: lowercase, with dashes if needed
  • Each module gets it's own Sass file components/_module.sass and style guide documentation.

Variable Naming Conventions

  • Choose semantic names for your variables.
  • all colour variables should be postfixed with -colour
  • Instead of describing what a variable looks like in the name, describe its function or purpose.
  • Variables that apply to specific sections should have a prefix like header- or footer- added
          
            // Bad
            $red: red;
            $yellow: yellow;

            // Better
            $brand-colour: red;
            $accent-colour: yellow;
